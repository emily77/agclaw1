# Repository Memory

## Stable Context
- **Issue 內容必須完整**：每個 Issue 在建立時就應提供清晰的目標、預期成果與必要背景說明，避免出現「僅有標題、內容空白」的資訊缺口。  
- **指派與標籤規範**：所有 Issue 必須指派負責人（Assignee）並加上適當的標籤（Label），以降低後續追蹤與協作成本。  
- **資料來源原則**：龍蝦們只會從 GitHub Issue 與 Comment 中抽取資訊，絕不直接複製原始 Issue 文字於手動筆記。  
- **手動筆記的角色**：`shared/manual.md` 只保存長期穩定規則、決策與限制，系統會讀取但不會覆寫此檔。  
- **快照範圍**：每日快照僅檢視最近 30 天、最多 100 個 Issue，超出範圍的資訊不會自動納入記憶。  
- **龍蝦工作流程**：每日由多隻龍蝦在「岩石縫隙」間爬行，逐一檢查 Issue，將可辨識的資訊寫入沙底，未能辨識的則標記為「資訊缺口」待主人補充。
