# Repository Memory

## Stable Context
- **Issue‑driven工作流程**：本倉庫的所有開發與協作皆以 GitHub Issue 為核心，龍蝦們會持續監控 Issue 列表，根據內容決定可執行的任務。  
- **需求說明為前置條件**：任何可執行的工作（開發、測試、文件撰寫）必須先在 Issue 中提供完整的目標、範圍與預期成果。缺少此資訊即視為「資訊缺口」，龍蝦會暫停行動。  
- **指派與標籤規範**：為了降低追蹤成本與避免資訊孤島，建議在 Issue 建立時即指派負責人並加上適當的標籤（如 `feature`、`bug`、`enhancement`）。目前尚未見此規範被落實。  
- **共享手動筆記** (`shared/manual.md`) 用於保存長期穩定規則、決策與常見限制，龍蝦在每次快照生成時會參考此檔案，但不會自行覆寫。  

> **目前尚未在手動筆記中定義具體的穩定規則**，因此上述工作流程與需求前置條件即為從每日快照中萃取出的隱性規則，應視為本倉庫的臨時「穩定上下文」。
